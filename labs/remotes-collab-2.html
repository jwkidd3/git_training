Remotes: Collaborating as a Team

Overview

We’re going to all work on one file together - we will do this simultaneously (after the instructor prepares the file) and then see what happens with our pull/merge requests.

The instructor will: create a readme.md file with a list of our names. The instructor will need to do this first.

Then students will: submit a pull/merge request to add their name to the readme.md file.

1. Is the Team Project set up?

Chase the instructor to get this done

  • Create a readme.md (or update it)
  • Add the right content
# Team Page

* [Ryan Morris](./ryan-morris.md)
* <-- Next person here

> Team as of 2021

2. Get up to date

Your task is to add your name to the readme.md file and submit that as a pull/merge request targeting the main branch.

In order to contribute to this file, you’ll need to start from an up-to-date main branch.

pull to get your main branch up to date:

git switch main
git pull

3. Do the work in a new branch

Start a new branch for the work you’re about to do:

git switch -c add-my-name-to-readme

⚠️ Remember to choose a unique branch name!

Do the work then push your branch up. Finally, open a new pull request:

# edit the readme
git add .
git commit -m "Added my name to the profile"
git push origin HEAD

⚠️ Need help with the edit are you making? If your name is “Jim Kirk”, you’d modify the readme.md to be something like this:

# Team Page

* [Ryan Morris](./ryan-morris.md)
* [Jim Kirk](./jim-kirk.md)
* <-- Next person here

> Team as of 2021

Once your PR is open, you’re all set.
