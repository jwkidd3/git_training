Stage & Commit Flow

Overview

You‚Äôre going to create a simple profile project to practice the basic git commands and the ‚Äústage & commit‚Äù workflow.

1. Initialize a new repository

Use init to start a new git project called my-profile.

üîé Without any arguments, git init will initialize a repository in your current directory.

This directory is going to be your root directory; all files and sub-directories within this top-level directory can be tracked as part of your project.

# make a directory to contain your project - in any way you like
mkdir my-profile

# initialize a repository inside that directory
cd my-profile
git init ./

üîé Alternatively you could have used git init my-profile to automatically create the folder and initialize a git project within it.

2. The first commit

Your project doesn‚Äôt have a history yet. If you run git log you‚Äôll get an error message. Let‚Äôs get at least one commit in the project.

Create an empty commit as your first commit.

# strange error message when there is no history..
git log

# commit with an empty changeset (you'll still need a message)
git commit --allow-empty

üîé You can also --allow-empty-message if you want to commit w/out providing a message, however this is not a good habit since messages help to describe the change that a commit contains.

Check the log, you should see the one commit you just created.

$> git log

commit 0584d0fce14736e93d736c106c31106f6f4bc138 (HEAD -> main)
Author: Ryan Morris <[email protected]>
Date:   Tue Aug 17 11:25:26 2021 -0700

    My First Commit

üîé It‚Äôs not necessary to start a project with an empty commit but it can be a handy way to begin a project that has no initial files.

3. Create a Profile file

Create a new, empy file, profile.md:

‚ùì You can create this as any file extension you wish, txt, md (markdown), etc.

# this creates a new file in the command line
# it is NOT a git command
touch profile.md

Check the status. git notices the new file, which will be untracked to start.

üîé Untracked files are not managed by git; git will avoid affecting them except for a very limited set of commands.

$> git status

On branch main

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	profile.md

Stage the change:

git add profile.md

Again, check the status:

$> git status

On branch main
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
	new file:   profile.md

üîé The file is now ‚Äústaged‚Äù and is in the list of files that will be committed.

Now, commit the change:

git commit

Finally, Check the status to confirm you don‚Äôt have any pending changes.

$> git status

On branch main
nothing to commit, working tree clean

üîé This is telling you you‚Äôre on a branch called main (our default branch), and there are no pending changes (no ‚Äúwork in progress‚Äù). Your ‚Äúworking tree‚Äù is the directory you‚Äôre working in.

4. Add a Title

Update your profile file so it has your name as a ‚Äútitle‚Äù within the contents.

example profile.md in markdown format:

# Ryan Morris

‚ùì Use any editor you like - such as your IDE, or a simple text editor. You are not using git‚Äôs core.editor for doing work‚Ä¶ that‚Äôs just for commit message collection.

Check your status, you should see some pending changes not yet staged.

Then stage your changes.

git add profile.md

Before you commit make another edit to the file, such as adding a list of favorite foods:

example profile.md in markdown format:

# Ryan Morris

Fav Foods:
* Pizza
* Murtabak

Check the status again:

$> git status

On branch main
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
	new file:   profile.md

Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
	modified:   profile.md

You should see that you have changes listed in two areas, your working directory and the staging area.

‚ùì If you were to commit now, only the staged changes would be put into the project history as a new commit.

Before you commit, view the pending changes in your working directory.

‚ùì Changes in your working directory are the edits you have not yet staged.

$> git diff

diff --git a/profile.md b/profile.md
index e0241d6..417d487 100644
--- a/profile.md
+++ b/profile.md
@@ -1 +1,5 @@
-# Ryan Morris
\ No newline at end of file
+# Ryan Morris
+
+Fav Foods:
+* Pizza
+* Murtabak
\ No newline at end of file

üîé git diff will show us work in progress that is in work working directory (not yet staged).

Now view your staged changes, using --staged:

$> git diff --staged

diff --git a/profile.md b/profile.md
index e69de29..e0241d6 100644
--- a/profile.md
+++ b/profile.md
@@ -0,0 +1 @@
+# Ryan Morris
\ No newline at end of file

Stage those pending changes which are in your working directory:

$> git add profile.md

$> git status

On branch main
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
	new file:   profile.md

üîé If you check the git diff now, what do you expect to see? You‚Äôll actually see no changes since you have nothing new/in progress within your working directory‚Ä¶ everything is in the staging area now.

Finally, commit your staged work.

git commit

üîé You don‚Äôt have to stage all your changes before you commit. You can have a ‚Äúdirty‚Äù working directory and still commit‚Ä¶ only your staged changes will be committed and your work in progress in the working directory will remain in place.

Check your log, formatting it so it shows only a single line per commit. Your commit ids and messages will be different than mine:

$> git log --oneline

50c79b2 (HEAD -> main) Updated my profile
5ca957c Added my profile
0584d0f My First Commit

5. Make another commit

Practice making a few more modifications to the profile.md file, staging and committing as you like. As you work, check your status and the log.

Bonus 1: Set up a difftool

This is optional.

You can configure git to use your preferred GUI client to view diffs, rather than displaying them in the command line.

Just as an example, here is how you might go about using Beyond Compare 4:

Not sure what tool to use? Try running git difftool --tool-help to see a list of tools that git knows how to work with.

git difftool --tool-help

You might also want to check if you already have a difftool set up:

git config diff.tool

  1. Make sure the application you want to use is installed

  2. Set up the tool with git

git config --global difftool.bc.path "c:/Program Files/Beyond Compare 4/bcomp.exe"
  1. Use it as a tool, on-the-fly:
git difftool -t bc HEAD^..HEAD
  1. Optionally instruct git to always use this ‚Äútool‚Äù by default:
git config --global diff.tool bc

Then you can invoke it like so:

git difftool HEAD^..HEAD

Bonus 2: Sub-directory

Create an empty sub-directory and see if you can commit the change:

mkdir tmp
git status

Git does not bother tracking empty directories. If you‚Äôd like to make git track this directory, create a file inside it:

touch tmp/empty
git add tmp/empty
git commit

The empty file is nothing specific to git, it‚Äôs a convention that some teams have adopted. You may also see teams use .gitkeep files to force git to track an empty directory.

Bonus 3: Explore .git

Every repository has a single, hidden .git directory at the top-level. Look inside:

ls -la
ls -la .git

The .git directory is a hidden directory managed by git, it contains your --local project config and all the data (every file you commit to the project) for this specific project.

$> ls -la .git/objects

# stuff...
.git/objects/92/c3b0c954b253d49aa4834707281ab3162aaa44
# more stuff...

If you want to view the contents of a git object, you can use cat-file on the full object identifier:

git cat-file -p 92c3b0c954b253d49aa4834707281ab3162aaa44

Git lightly compresses files and stores every version of every file within this ‚Äúobject database‚Äù.

Try using cat-file to view information about your last commit, ex:

$> git cat-file -p HEAD

tree 36ca93750277df9f79a7209bbc74e9189dce35d2
author RM 1630967329 -0400
committer RM 1630967329 -0400

First Commit

What is the tree id mentioned here? Try using cat-file on that identifier as well.

üîé A tree, which represents a directory listing, is one of the four object types git stores in its database, alongside blob, commit and tag.
