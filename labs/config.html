3. Essential Configs

Make sure you have both your user.name and user.name configured:

$> git config user.name
Michael Scott

$> git config user.email
[email protected]

If you don‚Äôt see a value echoed back then you will need to set the config values:

git config --global user.name "Michael Scott"
git config --global user.email "[email protected]"

üîé Git has three config levels: --system, --global and --local.

Windows tends to favor --system instead of --global.

Tip: If using --global fails because the ‚Äúfile does not exist‚Äù, use --system instead.

Finally, list all your configs:

git config --list

This will list configs based on your location, meaning, your current directory. git is a context-aware application; directory in which you run git will affect its behavior.

git config --list will use local, global or system, depending on what is available in that context.

git config --list --global will explicitly list only what is in the global config file.

4. Set an Editor (Optional)

As you create commits with Git you‚Äôll be prompted to type in a message describing the commit. Git will use vim as its default editor when it asks for a message, however you can customize this if you wish.

If you aren‚Äôt a vim fan then I recommend setting this to another editor of your choice.

üí° You should choose a text editor that can be opened and closed quickly. Your IDE (ex: JetBrains) is not appropriate for this kind of usage.

If desired, change the core.editor value.

Here are some popular options‚Ä¶ if you don‚Äôt see the editor you want to use then do a quick google search for your text editor + ‚Äúgit core.editor‚Äù.

# Windows - Notepad++
git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"

# Windows - Notepad++ in x86
git config --global core.editor "'C:/Program Files (x86)/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"

# Sublime Text
#
# Additional Requirement:
# This assumes you have `sublime text` installed _and_ you've set up the `subl` alias
# per [these instructions](https://www.sublimetext.com/docs/command_line.html)
#
# Or use homebrew:

git config --global core.editor "subl -n -w"

# BBEdit (Previously Text Wrangler)
git config --global core.editor "bbedit -w"

# VS Code
#
# Additional Requirement:
# This requires you also enable the "VS Code Shell Command", via the command-pallete
# Ex: cmd-shift-p, then search "Shell", "Enable Shell Command". 
# This will make it so you can execute "code" via the command line
git config --global core.editor "code --wait"

# Windows Notepad
git config --global core.editor notepad

# Atom
git config --global core.editor "atom --wait"

# Back to the default, VIM
git config --global core.editor vim

All configurations are stored in plaintext format. You can ask git to edit the config file with git config --edit and it will use the core.editor value.

Test your editor settings by using git to edit the configuration file:

git config --global --edit

# or, if using the system level...
git config --system --edit

5. Other Helpful Configs (Optional)

core.pager

If you want to disable the ‚Äúpager‚Äù, you can turn it off by providing an empty string to git as the core.pager value.

# optional: turn it off
git config --global core.pager ""

# optional: turn it (back) on (default)
git config --global core.pager less

# optional: turn it off for just the branch listing
git config --global pager.branch ""

help.autoCorrect

# only if you're on git 2.34 or above...
git config --global help.autoCorrect "prompt"

alias.graph

Do you want a helpful alias for graphing log output?

git config --global alias.graph "log --oneline --graph --all --decorate"
